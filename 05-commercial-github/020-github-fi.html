5.2. GitHub:FI¶

想要在企业内部架设自己的GitHub服务？答案就是GitHub:FI。

图5-10：GitHub:FI

GitHub:FI是GitHub Firewall Install的缩写，含义为在企业的防火墙内部架设GitHub服务。对于那些出于隐私或法律原因而不能将代码托管到第三方平台中的企业，用GitHub:FI在企业内部架设专有GitHub服务，可以在保证数据安全的同时，使用GitHub进行团队协同。

因为GitHub:FI采用了和GitHub近乎相同的代码，因此会使用GitHub，就会使用GitHub:FI。至于GitHub:FI的部署，因为采用了BitRock的打包和部署方案 [1] ，所以可以非常方便的在Linux平台（Ubuntu、Debian或Gentoo）上安装。

GitHub:FI的网站 [2] 中提供了演示录像，详细介绍了GitHub:FI的安装和使用。图5-11就是演示录像中介绍如何安装的画面。

图5-11：GitHub:FI演示画面

GitHub:FI的购买或试用，访问GitHub:FI网站并和销售人员联系 [3] 。

[2]http://fi.github.com/tour.html
